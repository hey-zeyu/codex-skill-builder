# 输出模式参考

用于设计 Codex skill 创建、审查和改造时的输出格式。

## Skill 蓝图

```markdown
## Skill 蓝图

### I/O 契约
- 输入：
- 输出：
- 触发场景：

### 目录结构
skill-name/
├── SKILL.md
├── references/   <如需要>
├── scripts/      <如需要>
└── assets/       <如需要>

### 工作流
1. ...
2. ...

### 需要用户准备
- 无 / <具体文件、密钥、文档>
```

## Skill 审查

```markdown
结论：<适合 / 改造后适合 / 不适合>

主要问题：
1. ...
2. ...

建议改法：
1. ...
2. ...

是否建议安装：
- <建议 / 暂不建议>
```

## 架构判断

```markdown
我判断这个 skill 应该这样拆：

- 复杂度：<简单/中等/复杂>
- 是否需要 scripts：<需要/不需要>，原因：
- 是否需要 references：<需要/不需要>，原因：
- 是否需要 assets：<需要/不需要>，原因：
- 主要安全边界：

建议目录结构：
...
```

## 错误输出

```markdown
无法完成：<一句话原因>

原因：
- ...

可以继续的方案：
1. ...
2. ...
```

## 完成输出

```markdown
已完成 <skill-name> 的 Codex 版改造。

改动重点：
- ...
- ...

验证：
- ...

位置：
- <path>
```
